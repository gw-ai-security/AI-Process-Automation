# Documentation Maintenance Skill

## Purpose

Keep project documentation aligned with implementation and scope boundaries.

## Responsibilities

- update README when startup paths or API contracts change
- keep architecture and phase docs current
- document known limitations and non-goals

## Rules

- write concise docs with recruiter-ready clarity
- prefer explicit scope statements over broad claims
- keep future plans high-level unless implemented

## Repo Touchpoints

- `README.md`
- `docs/`
- `CHANGELOG.md`
