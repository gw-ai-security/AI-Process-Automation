# Agents Scaffold

This directory defines project-side agent roles, reusable skills, and templates for maintainability. It is documentation and configuration scaffold only.

## Purpose

- clarify responsibilities across backend, documentation, QA, and orchestration work
- document allowed and forbidden scope to reduce overreach
- provide reusable project skills for consistent contributions

## Boundaries

- this is not an autonomous runtime
- there is no in-repo agent execution engine
- manifests are human-readable governance artifacts for future collaboration

## Phase 1 Intent

Keep the structure professional and explicit without turning the project into an AI platform.
